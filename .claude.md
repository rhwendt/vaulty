# Vaulty: Context Memory System for Claude

This repository is a context memory system designed to help Claude maintain consistent behavior and reference best practices across different tasks and projects. It contains memory files, agent prompts, and project structures that Claude should use when assisting with various activities.

## How This System Works

1. **User Configuration** (`config.md`): Personal settings and preferences (ALWAYS CHECK FIRST)
2. **Memory Files** (`memory/`): Store best practices and guidelines for different domains
3. **Agent Prompts** (`agents/`): Specialized agent personas that reference memory files
4. **Projects** (`projects/`): Structured project management with tasks and tracking

## Specialized Agents

Claude should invoke specialized agents based on the task at hand. Each agent has expertise in a specific domain and references relevant memory files.

### Git Agent (`agents/git-agent.md`)

**When to Use**: Git operations, version control, commits, pushes, pull requests

**Trigger Patterns**:
- "commit these changes"
- "push to git"
- "create a pull request"
- "create a branch"
- "merge this code"
- When code changes are complete and need version control

**What It Does**:
- Reads `memory/git-workflow.md` for best practices
- Follows commit message conventions
- Creates properly formatted PRs
- Manages branches according to standards
- Ensures no secrets are committed

### Developer Agent (`agents/developer-agent.md`)

**When to Use**: Writing code, implementing features, fixing bugs

**Trigger Patterns**:
- "write a script/function/class/program"
- "implement this feature"
- "fix this bug"
- "create a [language] solution"
- "develop functionality for"
- When user needs code written

**What It Does**:
- Reads `memory/architecture-design.md` for patterns
- Reads `memory/code-review.md` for quality standards
- Writes clean, tested code
- Follows SOLID principles
- Hands off to Tester Agent for testing
- Hands off to Auditor Agent for review

**Collaboration**: Developer ‚Üí Tester ‚Üí Auditor ‚Üí Git

### Tester Agent (`agents/tester-agent.md`)

**When to Use**: Writing tests, running test suites, checking coverage

**Trigger Patterns**:
- "write tests for"
- "test this code"
- "run the tests"
- "check test coverage"
- After Developer Agent completes code
- Before Auditor Agent reviews

**What It Does**:
- Reads `memory/testing-qa.md` for testing standards
- Writes unit, integration, and e2e tests
- Runs test suites and reports results
- Ensures coverage meets standards (>70%)
- Reports failures back to Developer Agent

**Collaboration**: Receives from Developer ‚Üí Returns to Developer if fails ‚Üí Sends to Auditor if passes

### Auditor Agent (`agents/auditor-agent.md`)

**When to Use**: Code review, security audits, quality checks

**Trigger Patterns**:
- "review this code"
- "audit this implementation"
- "check for security issues"
- After Tester Agent confirms tests pass
- Before Git Agent commits code

**What It Does**:
- Reads `memory/code-review.md` for review standards
- Reads `memory/testing-qa.md` for coverage requirements
- Checks for security vulnerabilities (OWASP Top 10)
- Verifies code quality and maintainability
- PASSES or FAILS code with detailed feedback
- If FAIL: sends back to Developer Agent with issues
- If PASS: approves for Git Agent to commit

**Collaboration**: Receives from Tester ‚Üí Returns to Developer if fails ‚Üí Sends to Git if passes

## Agent Collaboration Workflow

### Complete Development Workflow

When user requests a feature implementation:

1. **Project Manager Agent** creates/updates project and task
2. **Architect Agent** designs high-level architecture (if major feature)
3. **Software Design Agent** designs code structure and patterns
4. **Developer Agent** implements the code
5. **Tester Agent** writes and runs tests
   - If tests FAIL ‚Üí back to Developer Agent
6. **Auditor Agent** reviews code for quality and security
   - If audit FAILS ‚Üí back to Developer Agent
7. **Git Agent** commits and pushes code
8. **Documentation Agent** updates documentation
9. **Deployment Agent** deploys to environments
10. **Project Manager Agent** updates task status to completed

### Quick Bug Fix Workflow

When user reports a bug:

1. **Debugger Agent** investigates and finds root cause
2. **Developer Agent** implements fix
3. **Tester Agent** adds regression test
4. **Auditor Agent** reviews fix
5. **Git Agent** commits fix
6. **Deployment Agent** deploys hotfix

## Examples

### Example 1: User asks to implement a feature
```
User: "Create a Python script to process CSV files"

Claude Actions:
1. Invoke Project Manager Agent to create/track task (optional for small tasks)
2. Invoke Software Design Agent to plan structure
3. Invoke Developer Agent:
   - Read memory/architecture-design.md for patterns
   - Write the Python script
   - Add error handling
4. Invoke Tester Agent:
   - Read memory/testing-qa.md
   - Write tests for the script
   - Run tests
5. Invoke Auditor Agent:
   - Read memory/code-review.md
   - Review for security and quality
   - Check for vulnerabilities
6. If approved, invoke Git Agent:
   - Read memory/git-workflow.md
   - Create commit with proper message
   - Push to branch
```

### Example 2: User asks to commit changes
```
User: "Commit these changes to git"

Claude Actions:
1. Invoke Git Agent:
   - Read memory/git-workflow.md
   - Run git status and git diff
   - Review changes
   - Check for secrets
   - Create clear commit message
   - Commit and push
```

### Example 3: User reports a bug
```
User: "The login function isn't working"

Claude Actions:
1. Invoke Debugger Agent:
   - Gather information
   - Reproduce the issue
   - Analyze logs/errors
   - Identify root cause
2. Invoke Developer Agent to fix:
   - Implement fix
   - Test locally
3. Invoke Tester Agent:
   - Add regression test
   - Run test suite
4. Invoke Auditor Agent:
   - Review fix
5. Invoke Git Agent:
   - Commit fix
6. Invoke Deployment Agent (if needed):
   - Deploy hotfix
```
